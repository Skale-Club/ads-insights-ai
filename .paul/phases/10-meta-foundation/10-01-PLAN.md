<objective>
## Goal
Create the database schema for the Meta Ads integration: `companies` table (business profile), `meta_connections` table (OAuth tokens), and `meta_accounts` table (ad accounts). All tables with RLS policies following the existing pattern.

## Purpose
These tables are the data foundation everything else in v1.3 builds on. No Meta edge function or UI can work without them. Must be done first.

## Output
- Migration file with 3 new tables + RLS policies
- supabase/config.toml updated with new edge function stubs
</objective>

<acceptance_criteria>

## AC-1: companies table
```gherkin
Given a logged-in user
When they INSERT into companies with name, cnpj (optional), website (optional), vertical (optional)
Then the row is created with user_id = auth.uid()
And other users cannot SELECT/UPDATE/DELETE it (RLS enforced)
```

## AC-2: meta_connections table
```gherkin
Given a logged-in user
When a meta-auth edge function INSERTs an access_token for their user_id
Then the row stores: user_id, access_token, token_type='long_lived', expires_at, meta_user_id, meta_user_name
And other users cannot read it (RLS enforced)
And there is a unique constraint on user_id (one Meta connection per user for now)
```

## AC-3: meta_accounts table
```gherkin
Given a logged-in user with a meta_connection
When meta-accounts edge function INSERTs their ad accounts
Then each row stores: user_id, account_id (Meta format: act_XXXXXXXXX), account_name, currency, account_status, is_selected
And only that user can read/write their accounts
```

## AC-4: config.toml stubs
```gherkin
Given supabase/config.toml
When new Meta edge functions are referenced
Then meta-auth, meta-accounts, meta-reports, meta-mutate, get-meta-cli-session all have verify_jwt = false entries
```

<task type="auto">
  <name>Task 1: Create migration file</name>
  <files>supabase/migrations/20260408100000_meta_foundation.sql</files>
  <action>
    Create the migration with these 3 tables:

    **companies:**
    ```sql
    CREATE TABLE companies (
      id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
      user_id UUID NOT NULL REFERENCES auth.users(id) ON DELETE CASCADE,
      name TEXT NOT NULL,
      cnpj TEXT,
      website TEXT,
      vertical TEXT, -- e.g. 'ecommerce', 'saas', 'agency', 'local_business'
      created_at TIMESTAMPTZ NOT NULL DEFAULT now(),
      updated_at TIMESTAMPTZ NOT NULL DEFAULT now(),
      UNIQUE(user_id)  -- one company per user for now
    );
    ALTER TABLE companies ENABLE ROW LEVEL SECURITY;
    CREATE POLICY "companies_owner" ON companies
      USING (user_id = auth.uid())
      WITH CHECK (user_id = auth.uid());
    ```

    **meta_connections:**
    ```sql
    CREATE TABLE meta_connections (
      id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
      user_id UUID NOT NULL REFERENCES auth.users(id) ON DELETE CASCADE,
      access_token TEXT NOT NULL,
      token_type TEXT NOT NULL DEFAULT 'long_lived',
      meta_user_id TEXT NOT NULL,
      meta_user_name TEXT,
      expires_at TIMESTAMPTZ NOT NULL,
      created_at TIMESTAMPTZ NOT NULL DEFAULT now(),
      updated_at TIMESTAMPTZ NOT NULL DEFAULT now(),
      UNIQUE(user_id)
    );
    ALTER TABLE meta_connections ENABLE ROW LEVEL SECURITY;
    CREATE POLICY "meta_connections_owner" ON meta_connections
      USING (user_id = auth.uid())
      WITH CHECK (user_id = auth.uid());
    ```

    **meta_accounts:**
    ```sql
    CREATE TABLE meta_accounts (
      id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
      user_id UUID NOT NULL REFERENCES auth.users(id) ON DELETE CASCADE,
      account_id TEXT NOT NULL,   -- format: act_XXXXXXXXX
      account_name TEXT NOT NULL,
      currency TEXT NOT NULL DEFAULT 'USD',
      account_status INTEGER NOT NULL DEFAULT 1, -- 1=ACTIVE, 2=DISABLED, etc.
      is_selected BOOLEAN NOT NULL DEFAULT false,
      created_at TIMESTAMPTZ NOT NULL DEFAULT now(),
      updated_at TIMESTAMPTZ NOT NULL DEFAULT now(),
      UNIQUE(user_id, account_id)
    );
    ALTER TABLE meta_accounts ENABLE ROW LEVEL SECURITY;
    CREATE POLICY "meta_accounts_owner" ON meta_accounts
      USING (user_id = auth.uid())
      WITH CHECK (user_id = auth.uid());
    ```

    Also add a trigger for `updated_at` on all 3 tables using the same pattern as existing migrations.
  </action>
