Implement two Deno edge functions: `meta-auth` (handles the Meta OAuth callback — exchanges code for long-lived token, stores in meta_connections) and `meta-accounts` (fetches the user's Meta Ad Accounts from Marketing API and upserts into meta_accounts table).

<task type="auto">
  <name>Task 1: Implement meta-auth edge function</name>
  <files>supabase/functions/meta-auth/index.ts</files>
  <action>
    Create a new Deno edge function that handles the Meta OAuth callback.

    The function is called as a GET redirect from Meta's OAuth dialog with ?code=X&state=USER_ID.

    **Flow:**
    1. Parse `code` and `state` (state = user_id passed in the initial OAuth URL) from query params
    2. Exchange code for short-lived token:
       ```
       GET https://graph.facebook.com/v20.0/oauth/access_token
         ?client_id=META_APP_ID
         &redirect_uri=META_REDIRECT_URI
         &client_secret=META_APP_SECRET
         &code={code}
       ```
    3. Exchange short-lived for long-lived token:
       ```
       GET https://graph.facebook.com/v20.0/oauth/access_token
         ?grant_type=fb_exchange_token
         &client_id=META_APP_ID
         &client_secret=META_APP_SECRET
         &fb_exchange_token={shortLivedToken}
       ```
    4. Fetch user info: `GET /me?fields=id,name&access_token={longLivedToken}`
    5. Calculate expires_at = now() + expires_in seconds
    6. Upsert into meta_connections using Supabase service role key:
       ```
       UPSERT { user_id: state, access_token, meta_user_id, meta_user_name, expires_at }
       ON CONFLICT (user_id) DO UPDATE SET ...
       ```
    7. Redirect to `${SITE_URL}/settings?meta_connected=true`

    On any error: redirect to `${SITE_URL}/settings?meta_error=${encodeURIComponent(errorMessage)}`

    Required env vars: META_APP_ID, META_APP_SECRET, META_REDIRECT_URI, SUPABASE_URL, SUPABASE_SERVICE_ROLE_KEY, SITE_URL

    Use `import { createClient } from "https://esm.sh/@supabase/supabase-js@2"` for the Supabase client (service role).
    Use `import { serve } from "https://deno.land/std@0.168.0/http/server.ts"`.

    Note: This function handles GET (redirect), so no CORS OPTIONS needed. Return HTTP 302 with Location header.
  </action>
  <verify>Read the file and confirm: GET handler, code exchange, long-lived token exchange, user info fetch, upsert, redirect.</verify>
  <done>AC-1 satisfied</done>
</task>

<task type="auto">
  <name>Task 2: Implement meta-accounts edge function</name>
  <files>supabase/functions/meta-accounts/index.ts</files>
  <action>
    Create a new Deno edge function that fetches and stores Meta Ad Accounts.

    Use the same structure as `google-ads-accounts/index.ts`:
    - Import `serve`, `z` from npm:zod@3.22.4
    - CORS headers
    - Zod request schema: `{ accessToken: z.string().min(1), userId: z.string().min(1) }`
    - On OPTIONS return cors headers

    **Flow:**
    1. Validate request body with Zod
    2. Call Meta API:
       ```
       GET https://graph.facebook.com/v20.0/me/adaccounts
         ?fields=id,name,currency,account_status,business
         &access_token={accessToken}
         &limit=50
       ```
    3. Map response `data` array to MetaAdAccount shape:
       ```ts
       { account_id: item.id, account_name: item.name, currency: item.currency, account_status: item.account_status }
       ```
    4. Upsert all accounts into meta_accounts table (service role client) for userId
       - On conflict (user_id, account_id): update account_name, currency, account_status
       - Set is_selected = true for first account if none currently selected
    5. Return `{ accounts: [...] }` with HTTP 200

    On Meta API failure: return Meta's HTTP status + `{ error: "..." }`.
    On Zod failure: return HTTP 400 + `{ error: "Invalid request: ..." }`.

    Meta account_status codes: 1=ACTIVE, 2=DISABLED, 3=UNSETTLED, 7=PENDING_RISK_REVIEW, 9=IN_GRACE_PERIOD, 100=PENDING_CLOSURE, 101=CLOSED, 201=ANY_ACTIVE, 202=ANY_CLOSED
  </action>
  <verify>Read the file and confirm: Zod validation, Marketing API call, upsert, proper HTTP status on errors.</verify>
  <done>AC-2, AC-3 satisfied</done>
